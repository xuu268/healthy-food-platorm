### 微服务详细设计

#### 1. 用户服务 (User Service)
```yaml
职责: 用户管理和健康数据管理
技术栈: Spring Boot + MySQL + Redis
核心功能:
  - 用户注册登录 (JWT认证)
  - 健康档案管理 (基础信息、健康指标)
  - 饮食偏好管理 (口味、禁忌、过敏)
  - 健康目标管理 (减重、控糖等)
  - 数据同步 (穿戴设备数据同步)

API接口:
  POST /api/users/register      # 用户注册
  POST /api/users/login         # 用户登录
  GET  /api/users/profile       # 获取用户资料
  PUT  /api/users/health        # 更新健康数据
  GET  /api/users/preferences   # 获取饮食偏好
```

#### 2. 菜品服务 (Dish Service)
```yaml
职责: 菜品信息管理和营养计算
技术栈: Spring Boot + MySQL + Redis
核心功能:
  - 菜品信息管理 (名称、图片、描述)
  - 营养成分管理 (热量、蛋白质、脂肪等)
  - 食材管理 (食材库、营养成分)
  - 菜品分类管理 (菜系、口味、烹饪方式)
  - 商家菜品管理 (商家自有菜品)

API接口:
  GET  /api/dishes              # 获取菜品列表
  GET  /api/dishes/{id}         # 获取菜品详情
  POST /api/dishes              # 创建菜品
  PUT  /api/dishes/{id}         # 更新菜品
  GET  /api/dishes/nutrition    # 计算菜品营养
```

#### 3. 推荐服务 (Recommendation Service)
```yaml
职责: 智能菜品推荐
技术栈: Python + Flask + Redis + Milvus
核心功能:
  - 基于内容的推荐 (营养特征匹配)
  - 协同过滤推荐 (用户相似度)
  - 深度学习推荐 (神经网络模型)
  - 实时推荐 (在线推理)
  - 推荐解释生成 (可解释AI)

API接口:
  POST /api/recommendations     # 获取推荐
  GET  /api/recommendations/explain  # 获取推荐解释
  POST /api/recommendations/feedback # 反馈推荐效果
  GET  /api/recommendations/history  # 获取推荐历史
```

#### 4. 营养服务 (Nutrition Service)
```yaml
职责: 营养计算和分析
技术栈: Python + FastAPI + MongoDB
核心功能:
  - 营养需求计算 (基于用户特征)
  - 营养摄入分析 (历史数据分析)
  - 营养缺口识别 (不足或过量)
  - 营养优化建议 (菜品组合优化)
  - 营养知识库管理 (营养学知识)

API接口:
  POST /api/nutrition/needs     # 计算营养需求
  POST /api/nutrition/analysis  # 分析营养摄入
  GET  /api/nutrition/gap       # 识别营养缺口
  POST /api/nutrition/optimize  # 优化营养摄入
  GET  /api/nutrition/knowledge # 获取营养知识
```

#### 5. 健康服务 (Health Service)
```yaml
职责: 健康评估和预测
技术栈: Python + TensorFlow/PyTorch
核心功能:
  - 健康风险评估 (慢性病风险)
  - 健康趋势预测 (指标变化预测)
  - 健康建议生成 (个性化建议)
  - 健康效果评估 (饮食效果评估)
  - 健康知识图谱 (健康关系网络)

API接口:
  POST /api/health/risk         # 评估健康风险
  POST /api/health/predict      # 预测健康趋势
  GET  /api/health/advice       # 获取健康建议
  POST /api/health/evaluate     # 评估健康效果
  GET  /api/health/knowledge    # 查询健康知识
```
